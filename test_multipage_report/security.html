What is Vertical Port Scan?

Definition

A vertical port scan targets multiple ports on the same host. Source 192.168.100.203 scanned 15 different ports, attempting to discover all available services.

Why is this a problem?

Port scans are reconnaissance activities that precede attacks. Attackers use scans to map your network, identify running services, and find vulnerabilities to exploit. While scanning itself may not be harmful, it indicates malicious intent and often precedes actual attacks.

How was it detected?

The tool detected 192.168.100.203 scanning 15 ports/hosts in a short time period, with connection attempts that match port scanning patterns (rapid SYN packets without completing handshakes).

Wireshark Filter

Use this filter in Wireshark to examine the specific traffic:

ip.src == 192.168.100.203 and tcp.flags.syn == 1

Recommended Actions

1. Apply the filter to see the scan traffic. 2. Use 'tcp.flags.syn == 1 and tcp.flags.ack == 0 and ip.src == 192.168.100.203' to see SYN packets. 3. Check for responses with 'tcp.flags.reset == 1' (closed ports) or 'tcp.flags.syn == 1 and tcp.flags.ack == 1' (open ports). 4. Identify which ports were targeted using 'Statistics ‚Üí Conversations ‚Üí TCP'. 5. Block source IP 192.168.100.203 at the firewall. 6. Review firewall rules to ensure only necessary ports are exposed. 7. Implement IDS/IPS to detect and block scanning. 8. Check if any services on scanned ports have known vulnerabilities.

üìù Filter Pattern Library

Basic Flow Filters

Traffic between two IPs:

ip.addr == 192.168.1.100 and ip.addr == 8.8.8.8

Specific TCP port:

ip.addr == 192.168.1.100 and tcp.port == 443

Specific UDP port:

ip.addr == 192.168.1.100 and udp.port == 53
Security Analysis Filters

Port scan detection (SYN packets from scanner):

ip.src == 203.0.113.50 and tcp.flags.syn == 1 and tcp.flags.ack == 0

SYN flood (DDoS) to target:

ip.dst == 192.168.1.1 and tcp.flags.syn == 1 and tcp.flags.ack == 0

UDP flood to target:

ip.dst == 192.168.1.1 and udp

ICMP flood:

ip.dst == 192.168.1.1 and icmp

ARP conflicts for specific IP:

arp and arp.src.proto_ipv4 == 192.168.1.100
DNS Analysis Filters

All DNS queries:

dns and dns.flags.response == 0

All DNS responses:

dns and dns.flags.response == 1

Query for specific domain:

dns.qry.name == "example.com"

DNS failures (NXDOMAIN):

dns.flags.rcode == 3

DNS to specific server:

ip.dst == 8.8.8.8 and dns
TLS/SSL Security Filters

All TLS traffic:

tls

TLS handshakes:

tls.handshake.type == 1

TLS to specific server:

tls and ip.addr == 93.184.216.34

Weak TLS versions (SSLv3, TLS 1.0, 1.1):

tls.handshake.version <= 0x0302
